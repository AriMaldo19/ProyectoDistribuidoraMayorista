Antes de crear el indice, la consulta recorria la tabla completa Detalle_Ventas para poder realizar el JOIN y los calculos. Esto generaba 1078 lecturas logicas y un tiempo de ejecucion de 360 ms.

Luego de crear un indice sobre la columna ID_Venta, el motor comenzo a utilizar esa estructura optimizada en lugar de recorrer toda la tabla.
Las lecturas bajaron a 662 y el tiempo de ejecucion se redujo a 302 ms.

Aunque la mejora no es extrema, demuestra como el uso de indices puede reducir el trabajo que realiza el motor de base de datos al ejecutar consultas analiticas.

Conclusion
La creacion del indice permitio reducir la cantidad de lecturas logicas y el tiempo de ejecucion.
Esto demuestra que el uso de indices puede optimizar consultas que trabajan con grandes volumenes de datos, especialmente cuando se realizan JOIN y agregaciones.


Conceptos clave
Scan: el motor recorre toda la tabla.
Indice: estructura que permite buscar datos mas rapido.
Lecturas logicas: cantidad de paginas que el motor tuvo que leer.
Si bajan las lecturas, la consulta esta trabajando de forma mas eficiente.
