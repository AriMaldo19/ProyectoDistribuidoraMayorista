Antes de crear el índice, la consulta recorría la tabla completa Detalle_Ventas para poder realizar el JOIN y los cálculos. Esto generaba 1078 lecturas lógicas y un tiempo de ejecución de 360 ms.

Luego de crear un índice sobre la columna ID_Venta, el motor comenzó a utilizar esa estructura optimizada en lugar de recorrer toda la tabla.
Las lecturas bajaron a 662 y el tiempo de ejecución se redujo a 302 ms.

Aunque la mejora no es extrema, demuestra cómo el uso de índices puede reducir el trabajo que realiza el motor de base de datos al ejecutar consultas analíticas.

🔹 Conclusión
La creación del índice permitió reducir la cantidad de lecturas lógicas y el tiempo de ejecución.

Conceptos clave
Scan: el motor recorre toda la tabla.
Índice: estructura que permite buscar datos más rápido.
Lecturas lógicas: cantidad de páginas que el motor tuvo que leer.
Si bajan las lecturas, la consulta está trabajando de forma más eficiente.
